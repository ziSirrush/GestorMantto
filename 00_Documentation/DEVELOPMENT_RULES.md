# Reglas de Desarrollo

-   Aiven es la única fuente de datos.
-   No usar Supabase.
-   Adaptar cualquier integración del equipo de desarrollo a Aiven.
-   Si una función requiere cambios de BD, definir primero las
    modificaciones en Aiven.
-   Entregar archivos completos.
-   No romper funcionalidades existentes.
-   Validar consola antes de publicar.
-   Validar GitHub y Netlify después de cada integración.
